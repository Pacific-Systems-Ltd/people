subscribe()
POST{subscription_url}
 ↳ Content-Type: application/ld+json
 ↳ Authorization: DPoP {token}

Body includes:
 • type — channel type URI
 • topic — resource URL
 • sendTo — webhook URL (webhook path)
Solid Server
Responds with 200 OK and JSON-LD confirming channel.

Error cases:
 • 415 → UnsupportedChannelError
 • 422 → InvalidSubscriptionError
⚠ Safety check (NOT-13) — validate_subscription_target(url) is called before POSTing. Rejects localhost, loopback (127.x), and RFC-1918 private addresses to block SSRF attacks.
↓ returns SubscriptionResult — channel_url, send_to / receive_from, topic
